# Contributing

Thanks for helping keep this list sharp.

## What we accept
- Books that improve real engineering decisions.
- A short, specific description that answers: **What problem does this book help me solve?**
- Clear tags: level + role + focus.

## What we reject
- Pirated content links (PDFs, shady mirrors).
- Vague “must-read” claims with no substance.
- Duplicate entries that add no new angle.

## Format rules
Add books to the right topic README using this structure:

### *Title* — Author
**Tags:** `level` `role` `focus`

**Why we recommend it:** 3–7 sentences.
**What you get:** 3–6 bullets.
**Good to know:** 1–3 sentences (limits, dated parts, prerequisites).

## Optional extras
If you add a book card (templates exist), link it from the topic page.
